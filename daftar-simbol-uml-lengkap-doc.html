Daftar Simbol Uml Lengkap Doc

Admin December 10, 2024

Daftar Simbol Uml Lengkap Doc - Visual Modeling Language (UML) adalah sebuah bahasa pengembangan yang digunakan untuk membuat model sistem berbasis pengguna yang sistematis dan inklusif. UML memberikan langkah-langkah yang sistematis untuk membangun model sistem dan mempermudah komunikasi diantara developer, designer, dan pengguna. Dalam artikel ini, kita akan membahas simbol UML lengkap dan cara penggunaannya.

Simbol UML Lengkap

Simbol UML terdiri atas berbagai elemen, seperti class, object, association, inheritance, dan lain-lain. Berikut adalah daftar simbol UML lengkap:

1. Klasifikasi

  • Class: digunakan untuk mewakili konsep dasar yang terdefinisi dalam sistem.
  • Interface: digunakan untuk mewakili kontrak antar-class.
  • Package: digunakan untuk mengelompokkan class dan interface.

2. Elemen Relasi

  • Association: digunakan untuk mewakili hubungan antara dua atau lebih class.
    • Navigabilitas: digunakan untuk menentukan arah navigasi dalam association.
  • Composition: digunakan untuk mewakili relasi komposisi antara class.
  • Aggregation: digunakan untuk mewakili relasi agregasi antara class.
  • Generalization: digunakan untuk mewakili relasi generalisasi antara class.
  • Realization: digunakan untuk mewakili relasi implementasi antara interface dan class.

3. Elemen Penjelasan

  • Note: digunakan untuk menambahkan komentar atau penjelasan pada model.
  • Comment: digunakan untuk menambahkan komentar pada model.
  • Constraint: digunakan untuk menentukan constraint pada model.

4. Elemen Interaksi

  • Use Case: digunakan untuk mewakili kegiatann pengguna pada sistem.
  • Actor: digunakan untuk mewakili pengguna pada sistem.
  • Use Case Realization: digunakan untuk mewakili implementasi use case.

5. Elemen State Machine

  • State: digunakan untuk mewakili keadaan pada sistem.
  • Transition: digunakan untuk mewakili perubahan state.
  • Initial State: digunakan untuk mewakili state awal pada sistem.

6. Elemen Activity

  • Activity: digunakan untuk mewakili kegiatan pada sistem.
  • Action: digunakan untuk mewakili langkah-langkah dalam activity.
  • Gateway: digunakan untuk menentukan konfigurasi jaringan pada activity.
  • Merge: digunakan untuk menggabungkan aktivitas.
  • ** fork**: digunakan untuk memungkinkan aktivitas bercabang.

7. Elemen Interaksi Sequence

  • Trace: digunakan untuk mewakili jalur interaksi antara aktor dan sistem.
  • ** Lifeline**: digunakan untuk mewakili jalur interaksi antara aktor dan sistem.

8. Elemen Object Flow

  • Object Flow: digunakan untuk mewakili aliran objek pada sistem.

9. Elemen Component

  • Component: digunakan untuk mewakili bagian-bagian dari sistem.
  • Port: digunakan untuk mewakili bagian-bagian dari component.

Cara Penggunaan Simbol UML

Simbol UML dapat digunakan untuk membuat model sistem yang sistematis dan inklusif. Berikut adalah cara penggunaan simbol UML:

  1. Mulai dengan membuat class diagram yang menunjukkan struktur sistem.
  2. Gunakan association dan composition untuk mewakili relasi antara class.
  3. Gunakan generalization dan realization untuk mewakili relasi antara class.
  4. Gunakan note dan comment untuk menambahkan komentar atau penjelasan pada model.
  5. Gunakan use case dan actor untuk mewakili kegiatann pengguna pada sistem.
  6. Gunakan state machine untuk mewakili keadaan pada sistem.
  7. Gunakan activity untuk mewakili kegiatan pada sistem.
  8. Gunakan interaction sequence untuk mewakili jalur interaksi antara aktor dan sistem.
  9. Gunakan object flow untuk mewakili aliran objek pada sistem.
  10. Gunakan component untuk mewakili bagian-bagian dari sistem.

Dalam artikel ini, kita telah membahas simbol UML lengkap dan cara penggunaannya. Simbol UML dapat membantu developer dan designer dalam membuat model sistem yang sistematis dan inklusif.
